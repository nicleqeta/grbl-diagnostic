GRBL BASIC Command Summary

One-page reference for the BASIC runner in this app, grouped by job type.

Line-numbered scripts Case-insensitive commands Numbers + strings

1. Data And Variables

  • LET name = expr: Assign expression result to a variable.
  • READ("$30") INTO name or LET name = READ("$30"): Read a GRBL setting from $$.
  • SETTING("$30") INTO name or LET name = SETTING("$30"): Alias for READ.
  • INPUT name, INPUT name, "Prompt", name = INPUT("Prompt"), LET name = INPUT("Prompt"): Ask user for input.
10 LET max = READ("$30")
    20 LET speed = INPUT("Enter speed")
30 LET pct = INT((speed / max) * 100)

2. Machine I/O Commands

  • SEND expr: Send a command string to GRBL. Expression is converted to text.
  • SEND expr TIMEOUT expr: Wait for terminal acknowledgement up to timeout ms.
  • SEND expr REQUIRE_OK: Fail statement if no ok/error/ALARM acknowledgement arrives.
  • SEND expr TIMEOUT expr REQUIRE_OK: Use both options together.
  • HOLD, RESUME, STATUS, SOFT_RESET: Realtime control commands.
10 LET speed = 900
    20 SEND "M3 S" & STR(speed) TIMEOUT 1200 REQUIRE_OK
    30 STATUS
    40 HOLD
    50 WAIT 500
    60 RESUME

3. Timing And State Waiting

  • WAIT expr: Delay for N ms.
  • WAIT_IDLE or WAIT_IDLE timeoutMs: Wait until machine is idle.
  • WAIT_STATE target or WAIT_STATE target TIMEOUT timeoutMs: Wait for specific state.
10 SEND "G0 X0 Y0" TIMEOUT 1500 REQUIRE_OK
20 WAIT_IDLE 20000
30 WAIT_STATE "idle" TIMEOUT 5000

4. User Messaging And Results

  • PRINT [expr]: Log a message to terminal. Omit the expression for a blank separator line.
  • CONFIRM expr: Show confirmation dialog. If canceled, script stops.
  • PAUSE: Pause script until resumed.
  • RESULT name, expr: Save a named result value.
  • REPORT: Print all saved results.
10 LET speed = 1200
    20 PRINT "Starting run"
    30 RESULT "target_speed", speed
    40 CONFIRM "Run with speed " & STR(speed) & "?"
    50 REPORT
    60 END

5. Program Flow

  • FOR name = start TO end [STEP step] and NEXT [name]: Counter loop with optional explicit step.
  • IF condition THEN GOTO line: Conditional jump. Also accepts IF cond THEN LET, IF cond THEN PRINT, IF cond THEN GOSUB, and IF cond THEN END as single-line inline statements.
  • GOTO line: Unconditional jump.
  • GOSUB line and RETURN: Subroutine call/return.
  • END: Stop script.
10 LET sum = 0
20 FOR i = 1 TO 5
30 LET sum = sum + i
40 NEXT i
50 PRINT "Sum = " & STR(sum)
60 END

6. Expression Functions

  • PI: Circle constant (3.141592653589793).
  • STR(expr): Convert value to string.
  • ABS(expr): Absolute value.
  • INT(expr): Integer truncation.
  • TRUNC(expr): Integer truncation (alias of INT).
  • CEIL(expr): Round up to next integer.
  • FLOOR(expr): Round down to previous integer.
  • SIGN(expr): Returns -1, 0, or 1 based on value sign.
  • ROUND(value[, digits]): Round to nearest integer or to N decimal places.
  • SQRT(expr): Square root.
  • HYPOT(x, y): 2D hypotenuse length.
  • EXP(expr): Exponential function (e^x).
  • LOG(expr): Natural logarithm (ln).
  • LN(expr): Alias for natural logarithm.
  • LOG10(expr): Base-10 logarithm.
  • CLAMP(value, min, max): Bound a value to a range.
  • MAX(a, b): Larger of two values.
  • MIN(a, b): Smaller of two values.
  • MOD(a, b) or a % b: Remainder (modulo).
  • SIN(angleDeg): Sine of an angle in degrees.
  • COS(angleDeg): Cosine of an angle in degrees.
  • TAN(angleDeg): Tangent of an angle in degrees.
  • SINH(value), COSH(value), TANH(value): Hyperbolic trig.
  • ASIN(value): Inverse sine, returns degrees.
  • ACOS(value): Inverse cosine, returns degrees.
  • ATAN(value): Inverse tangent, returns degrees.
  • ATAN2(y, x): Inverse tangent with quadrant handling, returns degrees.
  • ASINH(value), ACOSH(value), ATANH(value): Inverse hyperbolic trig.
  • RAD(degValue): Convert degrees to radians.
  • DEG(radValue): Convert radians to degrees.
  • RND(max): Random integer from 1 to max.
  • GCODE_PARAM(key[, fallback]): Read cached CNC-style parameters such as #100 or #<_x> with optional default fallback.
  • STATE() or STATE: Current machine state string.
  • BENCH_LAST_MS(): Latest completed BENCH START .../BENCH END ... batch duration captured by the app.
10 LET dx = 3
20 LET dy = 4
30 LET len = HYPOT(dx, dy)
40 LET ang = ATAN2(dy, dx)
50 LET angRad = RAD(ang)
60 LET angDeg = DEG(angRad)
70 LET xr = ROUND(len * COS(angDeg), 3)
80 LET safeFeed = CLAMP(5000, 200, 3000)
90 LET lane = 17 % 5
100 PRINT "Angle(deg): " & STR(angDeg)
110 PRINT "Projected X: " & STR(xr)
120 PRINT "Safe feed: " & STR(safeFeed)
130 PRINT "Lane: " & STR(lane)
140 PRINT "State: " & STATE()
150 END

7. Import Header Format

Plain-text BASIC imports can start with a short metadata header before the first numbered BASIC line. The importer reads only top-of-file REM header lines, then treats the rest as runnable BASIC.

  • REM TITLE: ..., REM VERSION: ..., and REM AUTHOR: ... set the matching app fields.
  • REM DESCRIPTION: ... can appear more than once. Repeated lines are joined into a multiline description in the app.
  • REM VAR name=value seeds a variable default. Placeholder names from the script or description are also added if they are missing.
  • If no title header exists, import falls back to the filename.
  • Clipboard paste detects G-code patterns and can convert macros/subroutines into step-by-step BASIC automatically.
REM TITLE: X Axis Acceleration Finder
REM VERSION: 1
REM AUTHOR: GitHub Copilot
REM DESCRIPTION: Raises X acceleration until the motor sounds bad.
REM DESCRIPTION: Suggests a safer fallback value near the edge.
REM VAR start_accel=200
REM VAR coarse_step=50
10 PRINT "X-axis acceleration test"
20 PRINT "Start at " & STR({start_accel})

8. Authoring Rules In This App

  • Validation: save is blocked while syntax errors exist.
  • Sharing: saved scripts are loaded by ?script=....
  • Overrides: matching query params override saved variable defaults.
  • Safety: runtime has instruction/time limits and stop controls.
  • Queue mode: GCode queue defaults to Fast buffered; switch to synchronous if your controller needs stricter ack pacing.

9. Example: X-Axis Acceleration Finder

This example ramps $120 upward, asks the operator after each pass whether the axis sounded bad or skipped, retests the edge with a finer step, and reports a conservative recommendation. $110 (max X feed rate) is temporarily raised so the test feedrate is not clamped during the run, then restored at the end.

After reporting results, the script compares the original and suggested $120 values and asks whether to apply the suggested value now or keep the original setting.

Note: the current BASIC runner supports typed INPUT prompts, not single-key capture, so this script asks for 1 when the test sounds bad and 0 when it still sounds clean.

REM TITLE: X Axis Acceleration Finder
REM AUTHOR: GitHub Copilot
  REM VERSION: 3
  REM DESCRIPTION: Raises X acceleration by {coarse_step} starting at {start_accel} until the motor sounds bad or skips, then refines by {fine_step} near the edge.
  REM DESCRIPTION: Compares original $120 to the found value and asks whether to apply the suggested setting or keep the original.
  REM VAR start_accel=800
  REM VAR coarse_step=200
  REM VAR fine_step=20
  REM VAR max_accel=4000
  REM VAR x_travel=400
  REM VAR x_feed=70000
  REM VAR repeats=1
REM VAR idle_timeout=20000
REM VAR settle_ms=250

10 PRINT "X-axis acceleration test"
20 PRINT "Clear the machine so X can move back and forth safely."
30 PRINT "This script will temporarily change $120 during the test."
40 LET proceed = INPUT("Type 1 to begin, or 0 to cancel")
50 IF proceed <> 1 THEN GOTO 9000
60 LET original_accel = SETTING("$120")
70 LET original_feed = SETTING("$110")
80 LET last_good = 0
90 LET first_bad = 0
100 LET fine_good = 0
110 LET fine_bad = 0
120 LET recommended = 0
130 LET current_accel = 0
140 LET answer = 0
150 LET max_tested_accel = 0
160 LET max_mid_speed_mm_min = 0
170 LET apply_choice = 0
180 PRINT "Original $120 = " & STR(original_accel)
190 PRINT "Original $110 = " & STR(original_feed)
200 PRINT "Temporarily setting feed speed high"
210 SEND "$110=60000" TIMEOUT 2000 REQUIRE_OK
220 SEND "G21" TIMEOUT 1000 REQUIRE_OK
230 SEND "G91" TIMEOUT 1000 REQUIRE_OK
240 PRINT "Starting coarse search..."

250 FOR test_accel = {start_accel} TO {max_accel} STEP {coarse_step}
260 LET current_accel = test_accel
270 LET max_tested_accel = MAX(max_tested_accel, current_accel)
280 GOSUB 1000
290 IF answer <> 0 THEN GOTO 410
300 LET last_good = current_accel
310 NEXT test_accel
320 PRINT "No failure found up to max tested value."
330 LET recommended = last_good
340 GOTO 710

410 LET first_bad = current_accel
420 PRINT "First bad coarse value = " & STR(first_bad)
430 IF last_good = 0 THEN GOTO 920
440 LET fine_good = last_good
450 PRINT "Starting fine search..."
460 FOR test_accel = last_good + {fine_step} TO first_bad STEP {fine_step}
470 LET current_accel = test_accel
480 LET max_tested_accel = MAX(max_tested_accel, current_accel)
490 GOSUB 1000
500 IF answer <> 0 THEN GOTO 610
510 LET fine_good = current_accel
520 NEXT test_accel
530 LET recommended = fine_good
540 GOTO 710

610 LET fine_bad = current_accel
620 LET recommended = fine_bad - {fine_step}
630 PRINT "First bad fine value = " & STR(fine_bad)
640 GOTO 710

710 LET max_mid_speed_mm_min = SQRT(max_tested_accel * {x_travel}) * 60
720 SEND "$110=" & STR(original_feed) TIMEOUT 2000 REQUIRE_OK
730 WAIT {settle_ms}
740 PRINT "Last good coarse value = " & STR(last_good)
750 PRINT "First bad coarse value = " & STR(first_bad)
760 PRINT "Last good fine value = " & STR(fine_good)
770 PRINT "First bad fine value = " & STR(fine_bad)
780 PRINT "Suggested conservative $120 = " & STR(recommended)
790 PRINT "Estimated peak speed at travel midpoint = " & STR(max_mid_speed_mm_min) & " mm/min"
800 RESULT "original_x_accel", original_accel
810 RESULT "last_good_coarse", last_good
820 RESULT "first_bad_coarse", first_bad
830 RESULT "last_good_fine", fine_good
840 RESULT "first_bad_fine", fine_bad
850 RESULT "suggested_x_accel", recommended
860 RESULT "max_tested_accel", max_tested_accel
870 RESULT "theoretical_midpoint_speed_mm_min", max_mid_speed_mm_min
880 REPORT
890 PRINT "Original $120 = " & STR(original_accel) & ", suggested $120 = " & STR(recommended)
900 INPUT apply_choice, "Use suggested $120 now? Enter 1 for yes, 0 for no"
905 IF apply_choice <> 1 THEN GOTO 960
910 SEND "$120=" & STR(recommended) TIMEOUT 2000 REQUIRE_OK
915 WAIT {settle_ms}
916 PRINT "Updated $120 to suggested value."
917 RESULT "applied_x_accel", recommended
918 GOTO 990

920 LET recommended = original_accel
930 PRINT "The starting acceleration already sounded bad."
940 PRINT "Keeping recommendation at original $120."
950 GOTO 710

960 SEND "$120=" & STR(original_accel) TIMEOUT 2000 REQUIRE_OK
970 WAIT {settle_ms}
980 PRINT "Kept original $120 setting."
985 RESULT "applied_x_accel", original_accel
990 END

1000 SEND "$120=" & STR(current_accel) TIMEOUT 2000 REQUIRE_OK
1010 WAIT {settle_ms}
1020 FOR j = 1 TO {repeats}
1030 SEND "G1 X" & STR({x_travel}) & " F" & STR({x_feed}) TIMEOUT 2000 REQUIRE_OK
1040 WAIT_IDLE {idle_timeout}
1050 SEND "G1 X-" & STR({x_travel}) & " F" & STR({x_feed}) TIMEOUT 2000 REQUIRE_OK
1060 WAIT_IDLE {idle_timeout}
1070 NEXT j
1080 INPUT answer, "Tested $120=" & STR(current_accel) & ". Enter 1 if bad, else 0"
1090 RETURN

9000 PRINT "Test cancelled."
9010 END

10. Example: Distance-Scaled Acceleration Plateau Map

This test answers the question: at what acceleration does my machine stop getting faster for a given move distance? It sweeps the X-axis acceleration setting ($120) upward from {start_accel} in steps of {accel_step} up to a ceiling of {max_accel}, while running a timed batch of back-and-forth moves at each level. The stroke starts at 1 mm and doubles each round — 1, 2, 4, 8… up to {max_distance} mm — so you get a curve showing how the best usable acceleration scales with travel distance.

For each distance, the script calculates how many passes are needed so the total travel stays close to {target_total_travel} mm, with a minimum of {min_batch_count} passes per batch. Before each batch it writes the new acceleration to $120, then waits {settle_ms} ms for the controller to absorb the change. Each move is issued at feedrate {x_feed} mm/min and the script waits up to {idle_timeout} ms for the machine to reach idle before continuing.

The script keeps track of the best batch time seen so far for each distance and the acceleration that produced it. It stops testing higher acceleration at that distance once two consecutive steps fail to improve that best-so-far time by at least {plateau_ms} ms, which acts as a practical diminishing-returns rule for that stroke length. When all distances are done (or the max is reached), $120 is restored to its original value.

Timing is measured by wrapping each batch in BENCH START … / BENCH END … markers. The app captures the wall-clock duration and exposes it back to the script via BENCH_LAST_MS(). The Preview window opens automatically on the first marker and switches to Benchmark view, where you can watch the best-acceleration-per-distance curve build in real time and copy the raw data as JSON for further analysis.

The default {plateau_ms} is deliberately conservative because host-side timing noise, USB latency, and controller-status polling can move a short batch by several milliseconds. If you are testing very short strokes, increase {target_total_travel} and/or {plateau_ms} so the stop condition reacts to a real trend rather than one noisy batch. 

REM TITLE: Distance-Scaled Acceleration Plateau Map
REM AUTHOR: GitHub Copilot
REM VERSION: 1
REM DESCRIPTION: Measures repeated X back-and-forth batches at increasing acceleration and doubling stroke lengths.
REM DESCRIPTION: Uses BENCH markers so the app can time each batch and plot best observed acceleration by distance in the Preview window.
REM VAR start_accel=400
REM VAR accel_step=100
REM VAR max_accel=4000
REM VAR max_distance=400
REM VAR x_feed=30000
REM VAR target_total_travel=80
REM VAR min_batch_count=4
REM VAR settle_ms=200
REM VAR idle_timeout=12000
REM VAR plateau_ms=20

10 PRINT "Distance-scaled acceleration plateau map"
20 PRINT "Clear the machine so X can move symmetrically around the current point."
30 LET proceed = INPUT("Type 1 to begin, or 0 to cancel")
40 IF proceed <> 1 THEN GOTO 9000
50 LET original_accel = SETTING("$120")
60 LET current_distance = 1
70 LET final_best_accel = original_accel
80 LET final_best_ms = 0
90 SEND "G21" TIMEOUT 1000 REQUIRE_OK
100 SEND "G91" TIMEOUT 1000 REQUIRE_OK

200 IF current_distance > {max_distance} THEN GOTO 8000
210 LET batch_count = MAX({min_batch_count}, INT({target_total_travel} / current_distance))
215 REM batch_count = number of passes per timed batch. {min_batch_count} ensures at least that many passes.
216 REM Each pass = 1 forward move + 1 backward move. Total moves in batch = batch_count * 2.
220 LET best_batch_accel = 0
230 LET best_batch_ms = 0
240 LET stagnant_steps = 0
250 PRINT "Distance " & STR(current_distance) & " mm with batch_count=" & STR(batch_count)

300 FOR test_accel = {start_accel} TO {max_accel} STEP {accel_step}
310 LET current_accel = test_accel
320 GOSUB 1000
330 LET current_ms = BENCH_LAST_MS()
340 PRINT "Measured distance=" & STR(current_distance) & " accel=" & STR(current_accel) & " ms=" & STR(ROUND(current_ms, 3))
350 LET improved = 0
360 IF best_batch_ms = 0 THEN LET improved = 1
370 IF improved = 0 AND current_ms + {plateau_ms} < best_batch_ms THEN LET improved = 1
380 IF improved = 1 THEN GOTO 520
390 LET stagnant_steps = stagnant_steps + 1
400 IF stagnant_steps < 2 THEN GOTO 600
410 PRINT "Plateau reached for distance=" & STR(current_distance)
420 GOTO 700

520 LET best_batch_ms = current_ms
530 LET best_batch_accel = current_accel
540 LET stagnant_steps = 0
550 PRINT "New best so far at distance=" & STR(current_distance) & " accel=" & STR(best_batch_accel) & " batch_ms=" & STR(ROUND(best_batch_ms, 3))

600 NEXT test_accel

700 PRINT "Best observed acceleration at distance=" & STR(current_distance) & " is " & STR(best_batch_accel) & " with batch_ms=" & STR(ROUND(best_batch_ms, 3))
710 LET final_best_accel = best_batch_accel
720 LET final_best_ms = best_batch_ms
730 LET current_distance = current_distance * 2
740 GOTO 200

1000 SEND "$120=" & STR(current_accel) TIMEOUT 2000 REQUIRE_OK
1010 WAIT {settle_ms}
1020 PRINT "BENCH START distance=" & STR(current_distance) & " accel=" & STR(current_accel) & " batch=" & STR(batch_count) & " repeat=1 feed=" & STR({x_feed})
1025 REM Batch loop: each iteration of FOR pass_index performs 2 moves (forward at 1040/1050, backward at 1060/1070)
1030 FOR pass_index = 1 TO batch_count
1040 SEND "G1 X" & STR(current_distance) & " F" & STR({x_feed}) TIMEOUT 2000 REQUIRE_OK
1050 WAIT_IDLE {idle_timeout}
1060 SEND "G1 X-" & STR(current_distance) & " F" & STR({x_feed}) TIMEOUT 2000 REQUIRE_OK
1070 WAIT_IDLE {idle_timeout}
1080 NEXT pass_index
1090 PRINT "BENCH END distance=" & STR(current_distance) & " accel=" & STR(current_accel) & " batch=" & STR(batch_count) & " repeat=1 feed=" & STR({x_feed})
1100 RETURN

8000 SEND "$120=" & STR(original_accel) TIMEOUT 2000 REQUIRE_OK
8010 WAIT {settle_ms}
8020 PRINT "Restored original $120 = " & STR(original_accel)
8030 PRINT "Last completed best distance=" & STR(current_distance / 2)
8040 PRINT "Last completed best accel=" & STR(final_best_accel)
8050 PRINT "Last completed best batch ms=" & STR(ROUND(final_best_ms, 3))
8060 RESULT "restored_x_accel", original_accel
8070 RESULT "last_distance_mm", current_distance / 2
8080 RESULT "last_best_accel", final_best_accel
8090 RESULT "last_best_batch_ms", ROUND(final_best_ms, 3)
8100 REPORT
8110 END

9000 PRINT "Test cancelled."
9010 END
