GRBL BASIC Command Summary

One-page reference for the BASIC runner in this app, grouped by job type.

Line-numbered scripts Case-insensitive commands Numbers + strings

1. Data And Variables

  • LET name = expr: Assign expression result to a variable.
  • READ("$30") INTO name or LET name = READ("$30"): Read a GRBL setting from $$.
  • SETTING("$30") INTO name or LET name = SETTING("$30"): Alias for READ.
  • INPUT name, INPUT name, "Prompt", name = INPUT("Prompt"), LET name = INPUT("Prompt"): Ask user for input.
10 LET max = READ("$30")
    20 LET speed = INPUT("Enter speed")
30 LET pct = INT((speed / max) * 100)

2. Machine I/O Commands

  • SEND expr: Send a command string to GRBL. Expression is converted to text.
  • SEND expr TIMEOUT expr: Wait for terminal acknowledgement up to timeout ms.
  • SEND expr REQUIRE_OK: Fail statement if no ok/error/ALARM acknowledgement arrives.
  • SEND expr TIMEOUT expr REQUIRE_OK: Use both options together.
  • HOLD, RESUME, STATUS, SOFT_RESET: Realtime control commands.
10 LET speed = 900
    20 SEND "M3 S" & STR(speed) TIMEOUT 1200 REQUIRE_OK
    30 STATUS
    40 HOLD
    50 WAIT 500
    60 RESUME

3. Timing And State Waiting

  • WAIT expr: Delay for N ms.
  • WAIT_IDLE or WAIT_IDLE timeoutMs: Wait until machine is idle.
  • WAIT_STATE target or WAIT_STATE target TIMEOUT timeoutMs: Wait for specific state.
10 SEND "G0 X0 Y0" TIMEOUT 1500 REQUIRE_OK
20 WAIT_IDLE 20000
30 WAIT_STATE "idle" TIMEOUT 5000

4. User Messaging And Results

  • PRINT expr: Log a message to terminal.
  • CONFIRM expr: Show confirmation dialog. If canceled, script stops.
  • PAUSE: Pause script until resumed.
  • RESULT name, expr: Save a named result value.
  • REPORT: Print all saved results.
10 LET speed = 1200
    20 PRINT "Starting run"
    30 RESULT "target_speed", speed
    40 CONFIRM "Run with speed " & STR(speed) & "?"
    50 REPORT
    60 END

5. Program Flow

  • FOR name = start TO end [STEP step] and NEXT [name]: Counter loop with optional explicit step.
  • IF condition THEN GOTO line: Conditional jump.
  • GOTO line: Unconditional jump.
  • GOSUB line and RETURN: Subroutine call/return.
  • END: Stop script.
10 LET sum = 0
20 FOR i = 1 TO 5
30 LET sum = sum + i
40 NEXT i
50 PRINT "Sum = " & STR(sum)
60 END

6. Expression Functions

  • PI: Circle constant (3.141592653589793).
  • STR(expr): Convert value to string.
  • ABS(expr): Absolute value.
  • INT(expr): Integer truncation.
  • ROUND(value, digits): Round to N decimal places (digits can be negative).
  • SQRT(expr): Square root.
  • SIN(angleDeg): Sine of an angle in degrees.
  • COS(angleDeg): Cosine of an angle in degrees.
  • TAN(angleDeg): Tangent of an angle in degrees.
  • ASIN(value): Inverse sine, returns degrees.
  • ACOS(value): Inverse cosine, returns degrees.
  • ATAN(value): Inverse tangent, returns degrees.
  • ATAN2(y, x): Inverse tangent with quadrant handling, returns degrees.
  • RAD(degValue): Convert degrees to radians.
  • DEG(radValue): Convert radians to degrees.
  • RND(max): Random integer from 1 to max.
  • STATE() or STATE: Current machine state string.
10 LET dx = 3
20 LET dy = 4
30 LET len = SQRT(dx * dx + dy * dy)
40 LET ang = ATAN2(dy, dx)
    50 LET angRad = RAD(ang)
    60 LET angDeg = DEG(angRad)
    70 LET xr = ROUND(len * COS(angDeg), 3)
    80 PRINT "Angle(deg): " & STR(angDeg)
    90 PRINT "Projected X: " & STR(xr)
    100 PRINT "State: " & STATE()
    110 END

7. Import Header Format

Plain-text BASIC imports can start with a short metadata header before the first numbered BASIC line. The importer reads only top-of-file REM header lines, then treats the rest as runnable BASIC.

  • REM TITLE: ..., REM VERSION: ..., and REM AUTHOR: ... set the matching app fields.
  • REM DESCRIPTION: ... can appear more than once. Repeated lines are joined into a multiline description in the app.
  • REM VAR name=value seeds a variable default. Placeholder names from the script or description are also added if they are missing.
  • If no title header exists, import falls back to the filename.
REM TITLE: X Axis Acceleration Finder
REM VERSION: 1
REM AUTHOR: GitHub Copilot
REM DESCRIPTION: Raises X acceleration until the motor sounds bad.
REM DESCRIPTION: Suggests a safer fallback value near the edge.
REM VAR start_accel=200
REM VAR coarse_step=50
10 PRINT "X-axis acceleration test"
20 PRINT "Start at " & STR({start_accel})

8. Authoring Rules In This App

  • Validation: save is blocked while syntax errors exist.
  • Sharing: saved scripts are loaded by ?script=....
  • Overrides: matching query params override saved variable defaults.
  • Safety: runtime has instruction/time limits and stop controls.
  • Queue mode: GCode queue defaults to Fast buffered; switch to synchronous if your controller needs stricter ack pacing.

9. Example: X-Axis Acceleration Finder

This example ramps $120 upward, asks the operator after each pass whether the axis sounded bad or skipped, retests the edge with a finer step, then restores the original $120 and $110 and reports a conservative recommendation. $110 (max X feed rate) is temporarily raised so the test feedrate is not clamped during the run.

Note: the current BASIC runner supports typed INPUT prompts, not single-key capture, so this script asks for 1 when the test sounds bad and 0 when it still sounds clean.

REM TITLE: X Axis Acceleration Finder
REM AUTHOR: GitHub Copilot
REM VERSION: 2
REM DESCRIPTION: Raises X acceleration until the motor sounds bad or skips, then retests near the edge.
REM DESCRIPTION: Restores original settings and reports a conservative $120 recommendation.
REM VAR start_accel=200
REM VAR coarse_step=50
REM VAR fine_step=10
REM VAR max_accel=2000
REM VAR x_travel=8
REM VAR x_feed=3000
REM VAR repeats=6
REM VAR idle_timeout=20000
REM VAR settle_ms=250

10 PRINT "X-axis acceleration test"
20 PRINT "Clear the machine so X can move back and forth safely."
30 PRINT "This script will temporarily change $120 during the test."
40 LET proceed = INPUT("Type 1 to begin, or 0 to cancel")
50 IF proceed <> 1 THEN GOTO 9000
60 LET original_accel = SETTING("$120")
70 LET original_feed = SETTING("$110")
80 LET last_good = 0
90 LET first_bad = 0
100 LET fine_good = 0
110 LET fine_bad = 0
120 LET recommended = 0
130 LET current_accel = 0
140 LET answer = 0
150 LET max_tested_accel = 0
160 LET max_mid_speed_mm_min = 0
170 PRINT "Original $120 = " & STR(original_accel)
180 PRINT "Original $110 = " & STR(original_feed)
190 PRINT "Temporarily setting feed speed high"
200 SEND "$110=60000" TIMEOUT 2000 REQUIRE_OK
210 SEND "G21" TIMEOUT 1000 REQUIRE_OK
220 SEND "G91" TIMEOUT 1000 REQUIRE_OK
230 PRINT "Starting coarse search..."

240 FOR test_accel = {start_accel} TO {max_accel} STEP {coarse_step}
250 LET current_accel = test_accel
260 IF current_accel > max_tested_accel THEN LET max_tested_accel = current_accel
270 GOSUB 1000
280 IF answer <> 0 THEN GOTO 400
290 LET last_good = current_accel
300 NEXT test_accel
310 PRINT "No failure found up to max tested value."
320 LET recommended = last_good
330 GOTO 700

400 LET first_bad = current_accel
410 PRINT "First bad coarse value = " & STR(first_bad)
420 IF last_good = 0 THEN GOTO 900
430 LET fine_good = last_good
440 PRINT "Starting fine search..."
450 FOR test_accel = last_good + {fine_step} TO first_bad STEP {fine_step}
460 LET current_accel = test_accel
470 IF current_accel > max_tested_accel THEN LET max_tested_accel = current_accel
480 GOSUB 1000
490 IF answer <> 0 THEN GOTO 600
500 LET fine_good = current_accel
510 NEXT test_accel
520 LET recommended = fine_good
530 GOTO 700

600 LET fine_bad = current_accel
610 LET recommended = fine_bad - {fine_step}
620 PRINT "First bad fine value = " & STR(fine_bad)
630 GOTO 700

700 LET max_mid_speed_mm_min = SQRT(max_tested_accel * {x_travel}) * 60
710 SEND "$120=" & STR(original_accel) TIMEOUT 2000 REQUIRE_OK
720 SEND "$110=" & STR(original_feed) TIMEOUT 2000 REQUIRE_OK
730 WAIT {settle_ms}
740 PRINT "Last good coarse value = " & STR(last_good)
750 PRINT "First bad coarse value = " & STR(first_bad)
760 PRINT "Last good fine value = " & STR(fine_good)
770 PRINT "First bad fine value = " & STR(fine_bad)
780 PRINT "Suggested conservative $120 = " & STR(recommended)
790 PRINT "Estimated peak speed at travel midpoint = " & STR(max_mid_speed_mm_min) & " mm/min"
800 RESULT "original_x_accel", original_accel
810 RESULT "last_good_coarse", last_good
820 RESULT "first_bad_coarse", first_bad
830 RESULT "last_good_fine", fine_good
840 RESULT "first_bad_fine", fine_bad
850 RESULT "suggested_x_accel", recommended
860 RESULT "max_tested_accel", max_tested_accel
870 RESULT "theoretical_midpoint_speed_mm_min", max_mid_speed_mm_min
880 REPORT
890 END

900 LET recommended = original_accel
910 PRINT "The starting acceleration already sounded bad."
920 PRINT "Leaving recommendation at the original $120 value."
930 GOTO 700

1000 SEND "$120=" & STR(current_accel) TIMEOUT 2000 REQUIRE_OK
1010 WAIT {settle_ms}
1020 FOR j = 1 TO {repeats}
1030 SEND "G1 X" & STR({x_travel}) & " F" & STR({x_feed}) TIMEOUT 2000 REQUIRE_OK
1040 WAIT_IDLE {idle_timeout}
1050 SEND "G1 X-" & STR({x_travel}) & " F" & STR({x_feed}) TIMEOUT 2000 REQUIRE_OK
1060 WAIT_IDLE {idle_timeout}
1070 NEXT j
1080 INPUT answer, "Tested $120=" & STR(current_accel) & ". Enter 1 if bad, else 0"
1090 RETURN

9000 PRINT "Test cancelled."
9010 END
