1. Data And Variables

  • LET name = expr: Assign expression result to a variable.
  • READ("$30") INTO name or LET name = READ("$30"): Read a GRBL setting from $$.
  • SETTING("$30") INTO name or LET name = SETTING("$30"): Alias for READ.
  • INPUT name, INPUT name, "Prompt", name = INPUT("Prompt"), LET name = INPUT("Prompt"): Ask user for input via a text field.
  • LET name = FORM(msgExpr, val1, val2): Show a button-choice dialog. The message is any expression — literal string, concatenation, variable references, etc. Each [Label] token in the rendered message becomes a button; the matching val argument is the numeric value returned when clicked. Numbers are implicitly converted to text so STR() is optional. Use \, to include a literal comma. Can also be written as INPUT(FORM(...)).

FORM() Button Dialog

FORM() shows a modal dialog with clickable buttons derived from [Label] tokens in the rendered message. The message argument is evaluated as a full expression at runtime, so you can embed variable values using string concatenation (&) or any other supported expression. Numbers are implicitly converted to text — STR() is optional inside the message. The remaining comma-separated arguments are the numeric values each button returns, matched left-to-right with the labels. Use \, to embed a literal comma in the message without splitting the argument list.

Two forms are supported:

  • LET x = FORM(...) — buttons only, no text field. The user must click a button. Use this for clear-cut choices where only the predefined values make sense.
  • LET x = INPUT(FORM(...)) — same buttons plus a text field. Buttons act as shortcuts that immediately resolve the dialog; the user can also type a custom value and press Enter. Use this when a free-form value should be possible alongside the presets.
REM FORM() — static message, buttons only
10 LET choice = FORM("Select test mode [Quick] [Full] [Cancel]", 1, 2, 0)
20 IF choice = 0 THEN GOTO 9000
30 PRINT "Mode: " & STR(choice)

REM FORM() — dynamic message (STR() optional; numbers auto-convert)
10 LET accel = SETTING("$120")
20 LET answer = FORM("Current $120 = " & accel & " mm/s². [Keep] [Double]", accel, accel * 2)
30 SEND "$120=" & STR(answer) TIMEOUT 2000 REQUIRE_OK

REM INPUT(FORM()) — text field shown; buttons are shortcuts
10 PRINT "Current $120 = " & STR(SETTING("$120"))
20 LET new_accel = INPUT(FORM("Set $120? [Keep Current] [Double It] — or type a value:", SETTING("$120"), SETTING("$120") * 2))
30 SEND "$120=" & STR(new_accel) TIMEOUT 2000 REQUIRE_OK

2. Machine I/O Commands

  • SEND expr: Send a command string to GRBL. Expression is converted to text.
  • SEND expr TIMEOUT expr: Wait for terminal acknowledgement up to timeout ms.
  • SEND expr REQUIRE_OK: Fail statement if no ok/error/ALARM acknowledgement arrives.
  • SEND expr TIMEOUT expr REQUIRE_OK: Use both options together.
  • SEND expr BUFFERED: Stream a normal line-based GRBL command using host-side RX buffer accounting. Buffered sends do not wait for machine idle after each line; use WAIT_IDLE after the final buffered command in a batch to flush all acknowledgements and then wait for motion to finish.
  • SENDBUFF expr: Shorthand for SEND expr BUFFERED — stream a command without waiting for ack.
  • SENDACK expr or SENDACK expr timeoutMs: Shorthand for SEND expr TIMEOUT timeoutMs REQUIRE_OK — send and wait for ack (default timeout 12 s).
  • HOLD, RESUME, STATUS, SOFT_RESET: Realtime control commands.

STATUS is also how the runtime learns Bf: capacity fields (planner slots and serial RX bytes). Call STATUS early in scripts that use buffered streaming so capacity is learned before large batches begin.

Rule of thumb for sender mode selection: serial buffer below ~512 bytes usually benefits from buffered sends; serial buffer above ~512 bytes is often as fast or faster with ACK-gated sends (SENDACK semantics). That larger-buffer pattern is common on modern ESP32-class firmware (for example FluidNC and many grblHAL targets).

10 LET speed = 900
    20 SEND "M3 S" & STR(speed) TIMEOUT 1200 REQUIRE_OK
    30 STATUS
    40 HOLD
    50 WAIT 500
    60 RESUME

3. Timing, State, And Incoming Lines

  • WAIT expr: Delay for N ms.
  • WAIT_IDLE or WAIT_IDLE timeoutMs: Wait until machine is idle.
  • WAIT_STATE target or WAIT_STATE target TIMEOUT timeoutMs: Wait for specific state.
  • LET name = WAIT_FOR_LINE(pattern, timeoutMs) or WAIT_FOR_LINE(pattern, timeoutMs) INTO name: Block until an incoming serial line matches pattern (regex or plain substring). Returns the matched line, or "" on timeout. Pass 0 for no timeout.
  • LET name = READ_LINE(timeoutMs) or READ_LINE(timeoutMs) INTO name: Return the very next incoming serial line regardless of content; "" on timeout.
10 SEND "G0 X0 Y0" TIMEOUT 1500 REQUIRE_OK
20 WAIT_IDLE 20000
30 WAIT_STATE "idle" TIMEOUT 5000
40 REM Wait for controller to ack a specific setting change
50 SEND "$30=1000"
60 LET ack = WAIT_FOR_LINE("ok|error", 2000)
70 PRINT "Got: " & ack

4. Connection Management

Scripts can open the serial port themselves using CONNECT, enabling fully scriptable connection sequences without the Connection sidebar. A script containing CONNECT may run even when no port is currently open.

  • CONNECT: Show the browser serial port picker and connect at 115200 baud with default settings.
  • CONNECT BAUD expr: Connect at the specified baud rate.
  • CONNECT BAUD expr DTR "mode": Apply a DTR signal mode after opening. Accept: "default", "pulse", "low", "high".
  • CONNECT BAUD expr DTR "mode" WAIT expr: Pause N ms post-connect (useful when a DTR pulse reboots the controller — gives it time to boot).
  • CONNECT … OBSERVE: Connect in observe-only mode; no commands are sent.
  • DISCONNECT: Close the current serial connection.

All keyword arguments are optional and combinable in any order. If the user cancels the port picker the script stops gracefully.

10 SET SEND_DELAY 0
20 CONNECT BAUD 115200
30 STATUS
40 LET reply = WAIT_FOR_LINE("Grbl|FluidNC|<", 3000)
50 IF reply = "" THEN GOTO 9000
60 PRINT "Connected: " & reply
70 END
9000 PRINT "No response — check cable."
9010 DISCONNECT
9020 END

5. Runtime Settings

Script-level settings tune timing and buffer parameters. All settings reset to their defaults at the end of every script run, so changes never leak into subsequent runs.

  • SET SEND_DELAY expr: Pause in ms appended after each SEND (default 100). Set to 0 to remove all inter-command delays — useful for benchmarks or when WAIT_FOR_LINE already manages pacing.
  • SET ACK_TIMEOUT expr: Default acknowledgement wait in ms (default 12000). Affects bare SEND without an explicit TIMEOUT, and WAIT_FOR_LINE / READ_LINE without an explicit timeout argument.
  • SET RX_BUFFER expr: Fallback RX buffer byte limit when no Bf: report has been observed (default 64). Controls the SEND … BUFFERED window. Once a real Bf: serial value is observed, runtime capacity estimates use observed values instead of this fallback.
  • SET SEND_MODE expr: Select buffered-intent routing policy: BUFFERED (always buffered), SENDACK (route buffered-intent sends to ack-gated behavior), or AUTO (use ACK-gated when observed serial buffer is at least 512 bytes, otherwise buffered). If no serial Bf: has been observed yet, AUTO stays conservative and uses buffered mode.
  • SET BANNER_TIMEOUT expr: Milliseconds to wait for a startup banner after CONNECT (default 3000).
  • SET READY_SETTLE expr: Post-connect idle settle time in ms (default 150).
  • SET SEQUENCE_DELAY expr: Inter-command spacing in ms during startup sequences (default 450).
  • SET STATUS_DELAY expr: Delay in ms between status polls (default 600).
  • RESET_SETTINGS: Restore all runtime settings to their defaults mid-script.

Why SEND_MODE AUTO exists: controller families differ a lot in serial buffering. Small-buffer controllers usually gain throughput from host-side buffered streaming, while large-buffer controllers often acknowledge so quickly that strict ACK-gated sends are equal or faster. AUTO keeps one script portable by choosing behavior from observed Bf: serial capacity instead of forcing authors to hardcode per-firmware send strategy.

REM Zero all delays for a fast benchmark run
10 SET SEND_DELAY 0
20 SET ACK_TIMEOUT 30000
30 STATUS
40 LET serial_cap = BF_SERIAL()
50 LET planner_cap = BF_PLANNER()
60 PRINT "Serial buffer: " & STR(serial_cap) & " bytes"
70 PRINT "Planner slots: " & STR(planner_cap)
80 SET SEND_MODE AUTO
90 FOR i = 1 TO 100
100 SEND "G0 X" & STR(i) BUFFERED
110 NEXT i
120 WAIT_IDLE 30000
130 RESET_SETTINGS
140 PRINT "Done — settings restored to defaults"
150 END

6. User Messaging And Results

  • PRINT [expr]: Log a message to terminal. Omit the expression for a blank separator line.
  • CONFIRM expr: Show confirmation dialog. If canceled, script stops.
  • PAUSE: Pause script until resumed.
  • RESULT name, expr: Save a named result value.
  • REPORT: Print all saved results.
  • stmtA : stmtB: Statement chaining on one numbered line. Execution is left-to-right; colons inside quoted strings are ignored.
10 LET speed = 1200
    20 PRINT "Starting run"
    30 RESULT "target_speed", speed
    40 CONFIRM "Run with speed " & STR(speed) & "?"
    50 REPORT
    60 END

7. Program Flow

  • FOR name = start TO end [STEP step] and NEXT [name]: Counter loop with optional explicit step.
  • IF condition THEN GOTO line: Conditional jump. Also accepts IF cond THEN LET, IF cond THEN PRINT, IF cond THEN GOSUB, and IF cond THEN END as single-line inline statements.
  • GOTO line: Unconditional jump.
  • GOSUB line and RETURN: Subroutine call/return.
  • END: Stop script.
10 LET sum = 0
20 FOR i = 1 TO 5
30 LET sum = sum + i
40 NEXT i
50 PRINT "Sum = " & STR(sum)
60 END

8. Expression Functions

  • PI: Circle constant (3.141592653589793).
  • STR(expr): Convert value to string.
  • ABS(expr): Absolute value.
  • INT(expr): Integer truncation.
  • TRUNC(expr): Integer truncation (alias of INT).
  • CEIL(expr): Round up to next integer.
  • FLOOR(expr): Round down to previous integer.
  • SIGN(expr): Returns -1, 0, or 1 based on value sign.
  • ROUND(value[, digits]): Round to nearest integer or to N decimal places.
  • SQRT(expr): Square root.
  • HYPOT(x, y): 2D hypotenuse length.
  • EXP(expr): Exponential function (e^x).
  • LOG(expr): Natural logarithm (ln).
  • LN(expr): Alias for natural logarithm.
  • LOG10(expr): Base-10 logarithm.
  • CLAMP(value, min, max): Bound a value to a range.
  • MAX(a, b): Larger of two values.
  • MIN(a, b): Smaller of two values.
  • MOD(a, b) or a % b: Remainder (modulo).
  • SIN(angleDeg): Sine of an angle in degrees.
  • COS(angleDeg): Cosine of an angle in degrees.
  • TAN(angleDeg): Tangent of an angle in degrees.
  • SINH(value), COSH(value), TANH(value): Hyperbolic trig.
  • ASIN(value): Inverse sine, returns degrees.
  • ACOS(value): Inverse cosine, returns degrees.
  • ATAN(value): Inverse tangent, returns degrees.
  • ATAN2(y, x): Inverse tangent with quadrant handling, returns degrees.
  • ASINH(value), ACOSH(value), ATANH(value): Inverse hyperbolic trig.
  • RAD(degValue): Convert degrees to radians.
  • DEG(radValue): Convert radians to degrees.
  • RND(max): Random integer from 1 to max.
  • GCODE_PARAM(key[, fallback]): Read cached CNC-style parameters such as #100 or #<_x> with optional default fallback.
  • BF_SERIAL(): Estimated serial RX capacity in bytes. Uses observed Bf: serial values; if none have been observed yet, returns the current SET RX_BUFFER fallback.
  • BF_PLANNER(): Estimated planner capacity in slots from observed Bf: planner values. Returns -1 when planner data is unavailable.
  • STATE() or STATE: Current machine state string.
  • BENCH_LAST_MS(): Wall-clock duration of the most recently completed BENCH START … BENCH END pair, in milliseconds. See Section 9: Benchmark Markers for the full marker protocol including BENCH META chart configuration.
  • CLOCK(): Current timestamp in milliseconds (performance.now()). Store the result in a variable to use as a lap-timer start point.
  • ELAPSED(t0): Milliseconds elapsed since t0 (a value previously captured by CLOCK()). Returns a number so it can be used in further math.
  • FORMAT_MS(ms): Format a millisecond number as a human-readable string — "456 ms" or "1.23 s". Returns a string; use only inside PRINT or string concatenation.
10 LET dx = 3
20 LET dy = 4
30 LET len = HYPOT(dx, dy)
40 LET ang = ATAN2(dy, dx)
50 LET angRad = RAD(ang)
60 LET angDeg = DEG(angRad)
70 LET xr = ROUND(len * COS(angDeg), 3)
80 LET safeFeed = CLAMP(5000, 200, 3000)
90 LET lane = 17 % 5
100 PRINT "Angle(deg): " & STR(angDeg)
110 PRINT "Projected X: " & STR(xr)
120 PRINT "Safe feed: " & STR(safeFeed)
130 PRINT "Lane: " & STR(lane)
140 PRINT "State: " & STATE()
150 END

9. Benchmark Markers

BENCH markers let a script time arbitrary sections of work and stream live results to the Preview window's Benchmark chart. All three directives are emitted as plain PRINT output — the app intercepts them during script execution.

BENCH META key=value …

Sets chart metadata for the current run. Send this once before the first BENCH START to configure how the Preview chart is drawn. Keys are case-insensitive. Available keys:

  • title=… — chart title shown in the Preview banner.
  • subtitle=… / description=… — secondary label text.
  • x=fieldName or xfield=fieldName — which field from BENCH START/END to map on the X axis. Defaults to sampleIndex (1-based order of completed samples).
  • y=fieldName or yfield=fieldName — which field to map on the Y axis. Defaults to elapsedMs.
  • series=fieldName or seriesfield=fieldName — field whose value splits samples into named series (one chart line per distinct value). Omit for a single series.
  • xlabel=… — X axis label shown on the chart.
  • ylabel=… — Y axis label shown on the chart. Defaults to Elapsed (ms).
  • chart=line or charthint=line — chart rendering hint (currently only line is supported).

BENCH START key=value …

Opens a timed window. The key-value pairs can be any arbitrary fields; they are attached to the resulting sample record and can be referenced in BENCH META axis mappings. The same set of keys is typically echoed in the matching BENCH END line.

BENCH END key=value …

Closes the timed window and records the sample. The app measures the wall-clock elapsed time as elapsedMs. All key-value fields from both the START and END lines are merged into the sample record, alongside elapsedMs, startedAtIso, and endedAtIso. The elapsed time is made available to the script via BENCH_LAST_MS(). The completed record is streamed to the Preview window immediately.

The Preview Benchmark chart auto-scales to all recorded points and renders each series in a distinct colour. The Copy JSON button in Preview exports the full record set with metadata in a self-describing format ready for downstream analysis or AI-assisted visualisation. 

REM Chart: accel on X, elapsed ms on Y, one line per distance
10 PRINT "BENCH META x=accel series=distance xlabel=Acceleration (mm/s²) ylabel=Duration (ms) title=Accel vs Speed"
20 LET accel = 400
30 PRINT "BENCH START accel=" & STR(accel) & " distance=10"
40 SEND "G1 X10 F5000" BUFFERED
50 SEND "G1 X-10 F5000" BUFFERED
60 WAIT_IDLE 8000
70 PRINT "BENCH END accel=" & STR(accel) & " distance=10"
80 LET ms = BENCH_LAST_MS()
90 PRINT "Took " & STR(ms) & " ms"

10. Import Header Format

Plain-text GCOM script imports can start with a short metadata header before the first numbered GCOM script line. Header lines use a ; prefix — the Save GCOM button generates this format automatically. The importer also accepts the legacy REM prefix for backward compatibility.

  • ; TITLE: ..., ; VERSION: ..., and ; AUTHOR: ... set the matching app fields.
  • ; DESCRIPTION: ... can appear more than once. Repeated lines are joined into a multiline description in the app.
  • Rendered descriptions support both simple placeholders like {feed} and inline expression placeholders like {{min_batch_count * 2}}.
  • ; VAR name=value seeds a variable default. Placeholder names from the script or description are also added if they are missing.
  • If no title header exists, import falls back to the filename.
  • Clipboard paste detects G-code patterns and can convert macros/subroutines into step-by-step GCOM automatically.
; TITLE: X-Axis Max travel Speed and Acceleration Finder
; VERSION: 1
; AUTHOR: GitHub Copilot
; DESCRIPTION: Raises X acceleration while sweeping fast X travel moves to find the highest reliable travel speed.
; DESCRIPTION: Suggests a conservative fallback near the edge and reports estimated midpoint travel speed.
; VAR start_accel=200
; VAR coarse_step=50
10 PRINT "X-axis acceleration test"
20 PRINT "Start at " & STR({start_accel})

11. Authoring Rules In This App

  • Validation: save is blocked while syntax errors exist.
  • Sharing: saved scripts are loaded by ?gcom=....
  • Overrides: matching query params override saved variable defaults.
  • Safety: runtime has instruction/time limits and stop controls.
  • Queue mode: GCode queue defaults to Fast buffered; switch to synchronous if your controller needs stricter ack pacing.

12. Example: X-Axis Max travel Speed and Acceleration Finder

This example ramps $120 upward while repeatedly running fast back-and-forth X travel moves, asks the operator after each pass whether the axis sounded bad or skipped, retests the edge with a finer step, and reports a conservative recommendation. $110 (max X feed rate) is temporarily raised so the test feedrate is not clamped during the run, then restored at the end.

After reporting results, the script compares the original and suggested $120 values and asks whether to apply the suggested value now or keep the original setting.

Note: this script uses button-based FORM() prompts so operator decisions are explicit during the test flow.

; TITLE: X-Axis Max travel Speed and Acceleration Finder
; AUTHOR: GitHub Copilot
; VERSION: 4
; DESCRIPTION: Raises X acceleration by {coarse_step} starting at {start_accel} while running fast X travel moves until motion sounds bad or skips, then refines by {fine_step}.
; DESCRIPTION: Estimates peak midpoint travel speed and asks whether to apply the suggested $120 setting or keep the original.
; VAR start_accel=800
; VAR coarse_step=200
; VAR fine_step=20
; VAR max_accel=4000
; VAR x_travel=400
; VAR x_feed=70000
; VAR repeats=1
; VAR idle_timeout=20000
; VAR settle_ms=250

10 PRINT "X-axis max travel speed and acceleration test"
20 PRINT "Clear the machine so X can move back and forth safely."
30 PRINT "This script will temporarily change $120 during the test."
40 LET proceed = FORM("X-axis travel test ready. [Begin Test] [Cancel]", 1, 0)
50 IF proceed <> 1 THEN GOTO 9000
60 LET original_accel = SETTING("$120")
70 LET original_feed = SETTING("$110")
80 LET last_good = 0
90 LET first_bad = 0
100 LET fine_good = 0
110 LET fine_bad = 0
120 LET recommended = 0
130 LET current_accel = 0
140 LET answer = 0
150 LET max_tested_accel = 0
160 LET max_mid_speed_mm_min = 0
170 LET apply_choice = 0
180 PRINT "Original $120 = " & STR(original_accel)
190 PRINT "Original $110 = " & STR(original_feed)
200 PRINT "Temporarily setting feed speed high"
210 SEND "$110=60000" TIMEOUT 2000 REQUIRE_OK
220 SEND "G21" TIMEOUT 1000 REQUIRE_OK
230 SEND "G91" TIMEOUT 1000 REQUIRE_OK
240 PRINT "Starting coarse search..."

250 FOR test_accel = {start_accel} TO {max_accel} STEP {coarse_step}
260 LET current_accel = test_accel
270 LET max_tested_accel = MAX(max_tested_accel, current_accel)
280 GOSUB 1000
290 IF answer <> 0 THEN GOTO 410
300 LET last_good = current_accel
310 NEXT test_accel
320 PRINT "No failure found up to max tested value."
330 LET recommended = last_good
340 GOTO 710

410 LET first_bad = current_accel
420 PRINT "First bad coarse value = " & STR(first_bad)
430 IF last_good = 0 THEN GOTO 920
440 LET fine_good = last_good
450 PRINT "Starting fine search..."
460 FOR test_accel = last_good + {fine_step} TO first_bad STEP {fine_step}
470 LET current_accel = test_accel
480 LET max_tested_accel = MAX(max_tested_accel, current_accel)
490 GOSUB 1000
500 IF answer <> 0 THEN GOTO 610
510 LET fine_good = current_accel
520 NEXT test_accel
530 LET recommended = fine_good
540 GOTO 710

610 LET fine_bad = current_accel
620 LET recommended = fine_bad - {fine_step}
630 PRINT "First bad fine value = " & STR(fine_bad)
640 GOTO 710

710 LET max_mid_speed_mm_min = SQRT(max_tested_accel * {x_travel}) * 60
720 SEND "$110=" & STR(original_feed) TIMEOUT 2000 REQUIRE_OK
730 WAIT {settle_ms}
740 PRINT "Last good coarse value = " & STR(last_good)
750 PRINT "First bad coarse value = " & STR(first_bad)
760 PRINT "Last good fine value = " & STR(fine_good)
770 PRINT "First bad fine value = " & STR(fine_bad)
780 PRINT "Suggested conservative $120 = " & STR(recommended)
790 PRINT "Estimated peak speed at travel midpoint = " & STR(max_mid_speed_mm_min) & " mm/min"
800 RESULT "original_x_accel", original_accel
810 RESULT "last_good_coarse", last_good
820 RESULT "first_bad_coarse", first_bad
830 RESULT "last_good_fine", fine_good
840 RESULT "first_bad_fine", fine_bad
850 RESULT "suggested_x_accel", recommended
860 RESULT "max_tested_accel", max_tested_accel
870 RESULT "theoretical_midpoint_speed_mm_min", max_mid_speed_mm_min
880 REPORT
890 PRINT "Original $120 = " & STR(original_accel) & ", suggested $120 = " & STR(recommended)
900 LET apply_choice = FORM("$120 was " & original_accel & ", suggested is " & recommended & ". [Apply Suggested] [Keep Original]", 1, 0)
905 IF apply_choice <> 1 THEN GOTO 960
910 SEND "$120=" & STR(recommended) TIMEOUT 2000 REQUIRE_OK
915 WAIT {settle_ms}
916 PRINT "Updated $120 to suggested value."
917 RESULT "applied_x_accel", recommended
918 GOTO 990

920 LET recommended = original_accel
930 PRINT "The starting acceleration already sounded bad."
940 PRINT "Keeping recommendation at original $120."
950 GOTO 710

960 SEND "$120=" & STR(original_accel) TIMEOUT 2000 REQUIRE_OK
970 WAIT {settle_ms}
980 PRINT "Kept original $120 setting."
985 RESULT "applied_x_accel", original_accel
990 END

1000 SEND "$120=" & STR(current_accel) TIMEOUT 2000 REQUIRE_OK
1010 WAIT {settle_ms}
1020 FOR j = 1 TO {repeats}
1030 SEND "G1 X" & STR({x_travel}) & " F" & STR({x_feed}) TIMEOUT 2000 REQUIRE_OK
1040 WAIT_IDLE {idle_timeout}
1050 SEND "G1 X-" & STR({x_travel}) & " F" & STR({x_feed}) TIMEOUT 2000 REQUIRE_OK
1060 WAIT_IDLE {idle_timeout}
1070 NEXT j
1080 LET answer = FORM("Tested $120=" & current_accel & ". [Bad or Skip] [Still Clean]", 1, 0)
1090 RETURN

9000 PRINT "Test cancelled."
9010 END

13. Example: Distance-Scaled Acceleration Plateau Map

This test answers the question: at what acceleration does my machine stop getting faster for a given move distance? It sweeps the X-axis acceleration setting ($120) upward from {start_accel} in steps of {accel_step} up to a ceiling of {max_accel}, while running a timed batch of back-and-forth moves at each level. The stroke starts at {min_distance} mm and doubles each round — {min_distance}, 2×{min_distance}, 4×{min_distance}, 8×{min_distance}… up to {max_distance} mm — so you get a curve showing how the best usable acceleration scales with travel distance.

For each distance, the script calculates how many passes are needed so the total travel stays close to {target_total_travel} mm, with a minimum of {min_batch_count} passes per batch. Before each batch it writes the new acceleration to $120, then waits {settle_ms} ms for the controller to absorb the change. Each pass issues two movement commands (forward, then reverse), so a timed batch sends {{batch_count * 2}} movement commands total, with a guaranteed minimum of {{min_batch_count * 2}} commands. Those commands are streamed with SEND ... BUFFERED and then a single WAIT_IDLE confirms that the final command has been accepted and the machine has actually returned to idle.

The script keeps track of the best batch time seen so far for each distance and the acceleration that produced it. It stops testing higher acceleration at that distance once two consecutive steps fail to improve that best-so-far time by at least {plateau_ms} ms, which acts as a practical diminishing-returns rule for that stroke length. When all distances are done (or the max is reached), $120 is restored to its original value.

Timing is measured by wrapping each batch in BENCH START … / BENCH END … markers. The app captures the wall-clock duration and exposes it back to the script via BENCH_LAST_MS(). The Preview window opens automatically on the first marker and switches to Benchmark view, where you can watch the acceleration-vs-distance curve build in real time and copy the raw data as JSON for further analysis. Each distance becomes its own series on the chart, with acceleration on the X axis and elapsed batch time on the Y axis.

The default {plateau_ms} is deliberately conservative because host-side timing noise, USB latency, and controller-status polling can move a short batch by several milliseconds. If you are testing very short strokes, increase {target_total_travel} and/or {plateau_ms} so the stop condition reacts to a real trend rather than one noisy batch. 

REM TITLE: Distance-Scaled Acceleration Plateau Map
REM AUTHOR: GitHub Copilot
REM VERSION: 3
REM DESCRIPTION: Measures repeated X back-and-forth batches at increasing acceleration and doubling stroke lengths from {min_distance} mm up to {max_distance} mm.
REM DESCRIPTION: Each timed batch streams batch_count * 2 movement commands, with a guaranteed minimum of {{min_batch_count * 2}} moves per acceleration test.
REM DESCRIPTION: Uses BENCH markers so the app can time each batch and chart batch duration vs acceleration per distance in the Preview window.
REM VAR start_accel=400
REM VAR accel_step=200
REM VAR max_accel=4000
REM VAR min_distance=1
REM VAR max_distance=32
REM VAR x_feed=10000
REM VAR target_total_travel=64
REM VAR min_batch_count=1
REM VAR settle_ms=200
REM VAR idle_timeout=12000
REM VAR plateau_ms=60

10 LET proceed = FORM("Distance-Scaled Acceleration Plateau Map\, clear the machine so X can move freely around the current point. Do not switch browser tabs during the test — tab switching throttles the script timer and will cause false high readings. [Begin Test] [Cancel]", 1, 0)
20 IF proceed <> 1 THEN GOTO 9000
30 PRINT "BENCH META x=accel series=distance xlabel=Acceleration (mm/s²) ylabel=Batch Duration (ms) title=Distance-Scaled Acceleration Plateau"
50 LET original_accel = SETTING("$120")
60 LET current_distance = {min_distance}
70 LET final_best_accel = original_accel
80 LET final_best_ms = 0
90 SEND "G21" TIMEOUT 1000 REQUIRE_OK
100 SEND "G91" TIMEOUT 1000 REQUIRE_OK

200 IF current_distance > {max_distance} THEN GOTO 8000
210 LET batch_count = MAX({min_batch_count}, INT({target_total_travel} / current_distance))
215 REM batch_count = number of passes per timed batch. {min_batch_count} ensures at least that many passes.
216 REM Each pass = 1 forward move + 1 backward move. Total moves in batch = batch_count * 2.
220 LET best_batch_accel = 0
230 LET best_batch_ms = 0
240 LET stagnant_steps = 0
250 PRINT "Distance " & STR(current_distance) & " mm with batch_count=" & STR(batch_count)

300 FOR test_accel = {start_accel} TO {max_accel} STEP {accel_step}
310 LET current_accel = test_accel
320 GOSUB 1000
330 LET current_ms = BENCH_LAST_MS()
340 PRINT "Measured distance=" & STR(current_distance) & " accel=" & STR(current_accel) & " ms=" & STR(ROUND(current_ms, 3))
350 IF best_batch_ms = 0 THEN GOTO 520
360 IF current_ms + {plateau_ms} < best_batch_ms THEN GOTO 520
370 LET stagnant_steps = stagnant_steps + 1
380 IF stagnant_steps < 2 THEN GOTO 600
390 PRINT "Plateau reached for distance=" & STR(current_distance)
400 GOTO 700

520 LET best_batch_ms = current_ms
530 LET best_batch_accel = current_accel
540 LET stagnant_steps = 0
550 PRINT "New best so far at distance=" & STR(current_distance) & " accel=" & STR(best_batch_accel) & " batch_ms=" & STR(ROUND(best_batch_ms, 3))

600 NEXT test_accel

700 PRINT "Best observed acceleration at distance=" & STR(current_distance) & " is " & STR(best_batch_accel) & " with batch_ms=" & STR(ROUND(best_batch_ms, 3))
710 LET final_best_accel = best_batch_accel
720 LET final_best_ms = best_batch_ms
730 LET current_distance = current_distance * 2
740 GOTO 200

1000 SEND "$120=" & STR(current_accel) TIMEOUT 2000 REQUIRE_OK
1010 WAIT {settle_ms}
1020 PRINT "BENCH START distance=" & STR(current_distance) & " accel=" & STR(current_accel) & " batch=" & STR(batch_count) & " repeat=1 feed=" & STR({x_feed})
1025 REM Batch loop: each pass performs 2 buffered moves; WAIT_IDLE runs once after the whole batch.
1030 FOR pass_index = 1 TO batch_count
1040 SEND "G1 X" & STR(current_distance) & " F" & STR({x_feed}) BUFFERED
1050 SEND "G1 X-" & STR(current_distance) & " F" & STR({x_feed}) BUFFERED
1060 NEXT pass_index
1070 WAIT_IDLE {idle_timeout}
1080 PRINT "BENCH END distance=" & STR(current_distance) & " accel=" & STR(current_accel) & " batch=" & STR(batch_count) & " repeat=1 feed=" & STR({x_feed})
1090 RETURN

8000 LET last_distance = current_distance / 2
8010 LET accel_delta = final_best_accel - original_accel
8020 LET accel_pct = 0
8030 IF original_accel = 0 THEN GOTO 8050
8040 LET accel_pct = ROUND((accel_delta / original_accel) * 100, 1)
8050 PRINT "Summary: longest distance=" & STR(last_distance) & " mm. Best observed $120=" & STR(final_best_accel) & " with batch_ms=" & STR(ROUND(final_best_ms, 3))
8060 PRINT "Previous $120=" & STR(original_accel) & " delta=" & STR(accel_delta) & " (" & STR(accel_pct) & "%)"
8070 IF accel_delta > 0 THEN GOTO 8100
8080 IF accel_delta < 0 THEN GOTO 8130
8090 GOTO 8160
8100 LET rationale = "Suggested is higher because it produced the shortest longest-stroke batch. This can improve throughput if motion and finish stay stable."
8110 GOTO 8170
8130 LET rationale = "Suggested is lower because higher acceleration stopped improving longest-stroke batch time. This can improve consistency and reduce stress."
8140 GOTO 8170
8160 LET rationale = "Suggested matches previous because no meaningful longest-stroke timing gain was found by changing acceleration."
8170 LET apply_accel = INPUT(FORM("Apply $120 update? Previous=" & original_accel & " Suggested=" & final_best_accel & " Delta=" & accel_delta & " (" & accel_pct & "%). Longest stroke=" & last_distance & " mm. Best batch=" & ROUND(final_best_ms, 3) & " ms. " & rationale & " [Apply Suggested] [Keep Previous]", final_best_accel, original_accel))
8180 SEND "$120=" & STR(apply_accel) TIMEOUT 2000 REQUIRE_OK
8190 WAIT {settle_ms}
8200 PRINT "Set $120 = " & STR(apply_accel)
8210 RESULT "applied_x_accel", apply_accel
8220 RESULT "original_x_accel", original_accel
8230 RESULT "last_distance_mm", last_distance
8240 RESULT "last_best_accel", final_best_accel
8250 RESULT "last_best_batch_ms", ROUND(final_best_ms, 3)
8260 RESULT "accel_delta", accel_delta
8270 RESULT "accel_delta_pct", accel_pct
8280 REPORT
8290 END

9000 PRINT "Test cancelled."
9010 END

14. Example: FluidNC Startup Flowchart

This script implements the FluidNC sender startup flowchart: open the port, send a status poll to wake the controller, wait for a banner (Grbl … or FluidNC …) or a live-state response (<Idle|…>), then read $30 to confirm two-way comms. A graceful DISCONNECT handles the no-response case.

SET SEND_DELAY 0 removes the default 100 ms inter-command pause. WAIT_FOR_LINE provides the blocking wait with its own timeout, making the explicit delay redundant.

REM TITLE: FluidNC Startup Flowchart
REM AUTHOR: GitHub Copilot
REM VERSION: 1
REM DESCRIPTION: Opens a serial port and runs the recommended FluidNC/GRBL sender startup
REM DESCRIPTION: sequence from wiki.fluidnc.com/en/support/gcode_senders.
REM DESCRIPTION: Polls with ? after connecting, waits up to {banner_timeout} ms for a banner
REM DESCRIPTION: or state response, then reads $30 to confirm two-way comms.
REM VAR baud=115200
REM VAR banner_timeout=3000
REM VAR settle_ms=200

10 REM Disable the default 100 ms inter-SEND pause; WAIT_FOR_LINE manages pacing
20 SET SEND_DELAY 0

30 REM Open the browser serial port picker and connect
40 CONNECT BAUD {baud}

50 REM ? is a realtime status poll — safe on any firmware state, provokes a response
60 STATUS

70 REM Wait for a startup banner (Grbl / FluidNC) or a live-state report (<Idle|...>)
80 LET first_line = WAIT_FOR_LINE("Grbl|FluidNC|<", {banner_timeout})
90 IF first_line = "" THEN GOTO 9000

100 PRINT "Controller responded: " & first_line
110 WAIT {settle_ms}

120 REM Read $30 to confirm full two-way (command + response) comms are working
130 LET max_rpm = READ("$30")
140 PRINT "Max spindle RPM ($30) = " & STR(max_rpm)
150 PRINT "Startup complete — add machine-specific steps below."
160 STATUS
170 END

9000 PRINT "No response within " & STR({banner_timeout}) & " ms."
9010 PRINT "Check baud rate ({baud}) and USB cable, then retry."
9020 DISCONNECT
9030 END
