What helps the AI today
  • Machine name and notes give the AI human context for the machine.
  • Axes tell the AI whether Z moves are allowed.
  • Unit mode and work offset shape preambles the AI generates.
  • Safe Z changes how rapid moves and tool-off sequences are described.
  • Peripherals add exact support commands like air assist on/off.
  • Commands and snippets are injected verbatim as exact GCOM examples.
  • Presets provide feed, power, rapid, and plunge defaults.
Stored but not injected today
  • Firmware is stored for reference but does not alter the AI prompt yet.
  • Tool type is documentation only. It does not infer controller settings like $32=1.
  • Bed width and height are stored for context but are not injected into the AI contract yet.
How to enter commands

Commands are plain GCOM examples, not templates and not line-numbered BASIC blocks.

Use unnumbered GCOM statements only. Put one statement on each line. Multiline blocks are valid for lifecycle commands like job_start and tool_off_sequence.

SET SEND_MODE AUTO SEND "G21" REQUIRE_OK SEND "G90" REQUIRE_OK SEND "G54" REQUIRE_OK SEND "M5 S0" REQUIRE_OK

When you save a command named job_start, the profiler sample uses that block instead of its synthesized preamble.

Base controller

Chooses the controller contract the profile is built on. This controls live guidance, starter boilerplate, recommended lifecycle examples, serial defaults, and which compatibility rules the AI will follow.

Machine info
  • Machine name: required. Used in the AI prompt as the machine identity.
  • Notes: injected directly into the AI prompt.
  • Unit mode: tells the AI whether to prefer G21 or G20.
  • Work offset: adds an explicit coordinate system like G54 to recommended preambles.
  • Axes: critical for lasers and other XY-only machines because it suppresses Z output.
  • Safe Z: only matters when Z exists and becomes guidance for rapid moves and tool-off behavior.
Peripherals

Air assist defaults to SEND "M8" REQUIRE_OK and SEND "M9" REQUIRE_OK when enabled and left blank. Saving these values makes them part of the AI prompt so generated scripts know how to turn support hardware on and off.

Commands

This is the highest-value section for AI quality. Saved commands are injected as exact examples, so the AI learns your preferred startup, shutdown, and motion idioms from real GCOM instead of guessing.

  • job_start: full preamble for this machine
  • job_end: full postamble or parking sequence
  • tool_on_sequence: spindle or laser on, usually with power
  • tool_off_sequence: spindle or laser off
  • rapid_move: representative positioning move
  • cut_move: representative cutting or marking move with feed
Snippets

Snippets are named reusable blocks that the AI sees as reference material. Use them for small machine rituals, fixture setup, or patterns you want the AI to reuse without pretending they are lifecycle hooks.

Presets

Presets give the AI numeric defaults for feed, power, rapid feed, and plunge feed. When these fields are blank, the profiler shows live defaults from the selected controller where available, or profiler fallbacks when no controller default exists.

Transport tuning

Transport tuning is sender-side only. It does not change controller firmware. These values help the host estimate buffer and planner capacity before live telemetry arrives.

Live preview and save flow

The machine test preview is a read-only sample that updates from the current form. Saving posts the profile to the worker and returns a shareable profile ID. Loading by ID restores a saved profile into the form.

Default sources
Controller defaults

These come from the selected controller profile, such as default feed, default power, serial baud, and ACK timeout. They are live-read only and never overwrite your form automatically.

Generated recommendations

The recommended lifecycle examples are built from the selected controller plus your current axes, unit mode, work offset, safe Z, peripherals, and presets. They are suggestions until you save them as real commands.

Profiler fallbacks

Some values do not exist in the controller profile today, such as rapid feed, plunge feed, RX buffer, and planner slots. The profiler shows these as explicit fallbacks so you can see what is assumed before you override them.
